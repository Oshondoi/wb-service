# WB Wildberries Analytics Service

Полноценный аналитический сервис для сбора максимум данных о товарах Wildberries с многоуровневыми фолбэками, публичными CSV для интеграции с Google Sheets и **модулем финансовых отчетов с мультикомпанией**.

- ✅ **Database**: Supabase PostgreSQL (cash_transactions, cash_debts, counterparties, cash_categories)
- ✅ **Authentication**: Работает (default: admin/tarelkastakan)
- ✅ **Financial Reports**: 71 колонка как на WB Partners сайте
- ✅ **Word Wrap**: Компактные заголовки с переносом слов
- ✅ **Text Alignment**: Все заголовки выровнены по левому краю
- ✅ **Auto-Sync**: Cron job каждую ночь в 3:30 AM (Бишкек) + автозагрузка при создании магазина (полная история с 2019)
- ✅ **LocalStorage**: Выбранный магазин и период запоминаются между обновлениями страницы
- ✅ **Routing Updated**: ДДС — главная страница (`/`), фин. отчёт — `/fin-report`

### Параметры API запросов:
- `businessId` — ID компании (опционально, если не указан — берётся первая активная)
- `dateFrom` / `dateTo` — диапазон дат для финансовых отчётов
- `nm` — артикул товара Wildberries
- `domain` — домен WB (ru/kz/kg)

### Пояснение ключевых полей
| Поле | Значение |
|------|----------|
| `price` | Минимальная найденная цена (конвертирована из *_U / 100) |
| `name` | Название товара |
| `brand` | Бренд товара |
| `sellerId` | Идентификатор продавца на WB |
| `storeName` | **ПОЛНОЕ юридическое лицо** продавца (live парсинг или база) |
| `category` | Категория товара (маппинг из subjectId) |
| `color` | Основной цвет товара |
| `rating` | Рейтинг товара (0..5) |
| `feedbacks` | Количество отзывов |
| `images` | Количество изображений (pics/images) |

| `mainImage` | URL основной картинки (webp с fallback) |

## Интеграция с Google Sheets
Минимальная формула для цены (артикул в ячейке C3):
```
=INDEX(IMPORTDATA("https://your-domain/wb-price-csv?nm=" & C3),2,1)
```
Название:
```
=INDEX(IMPORTDATA("https://your-domain/wb-price-csv?nm=" & C3),2,2)
```
Если нужно массовое обновление по диапазону артикулов — используйте Google Apps Script и цикл, дергающий `/wb-price-csv`.

## Логика извлечения цены
Собираются кандидаты из полей:
```
salePriceU, clientSalePriceU, basicPriceU, priceU, fullPriceU,
sizes[].price.{basic,product,total}, sizes[].{salePriceU, priceU}
```
Берётся минимальное положительное значение → делится на 100.

## Изображения
`mainImage` отдаётся через `/wb-image`, который перебирает basket‑хосты и кэширует найденный домен. Если изображение недоступно — возвращается SVG‑заглушка.
Можно напрямую запросить через прокси:
```
/wb-image?nm=307325849&pic=1
```

## 📈 Модуль финансовых отчетов ✅ РАБОТАЕТ

**Полноценный функционал** для комплексного учета финансов продаж на Wildberries:

### Реализованные возможности:
- ✅ **Интеграция с WB Statistics API** — прямое подключение через API ключи
  - Endpoint `/api/v5/supplier/reportDetailByPeriod` — полный финансовый отчет (82 поля)
  - Endpoint `/api/v1/supplier/sales` — данные о продажах
  - Endpoint `/api/v1/supplier/orders` — данные о заказах
- ✅ **Три типа отчетов в модальных окнах**:
  - **📈 Финансовый отчет** — детальный отчет со всеми 82 полями WB (как в личном кабинете)
  - **💰 Продажи** — сгруппированный отчет по уникальным артикулам с агрегацией количества
  - **📦 Заказы** — полный список заказов с детализацией по статусам
- ✅ **Расчет финансовых показателей**:
  - Общая выручка (retail_amount)
  - Комиссия WB (ppvz_sales_commission)
  - Логистика и расходы (delivery_rub + storage_fee + acquiring_fee + penalty + deduction + acceptance)
  - Чистая прибыль (К перечислению — ppvz_for_pay)
  - Расчет добытка после комиссий
- ✅ **5 информационных карточек**:
  - Общая выручка
  - Комиссия WB
  - Логистика + расходы
  - К перечислению (чистая прибыль)
  - Чистая прибыль (расчет добытка)
- ✅ **Фильтрация по датам** — выбор произвольного периода
- ✅ **Динамические заголовки таблицы**:
  - 82 колонки для финансового отчета
  - 10 колонок для отчета по продажам (артикул, предмет, бренд, кол-во, выручка, комиссия, логистика, прибыль, средняя цена, склад)
- ✅ **Sticky header** — заголовок таблицы остается видимым при прокрутке
- ✅ **Безопасное хранение API ключей** — файловое хранение (wb-api-key.txt в .gitignore), httpOnly cookies
- ✅ **Группировка продаж по артикулам** — каждый nmId встречается 1 раз с суммированием количества
- ✅ **Сортировка** — автоматическая сортировка продаж по количеству (от большего к меньшему)
- ✅ **Модальный интерфейс** — отчеты открываются в модальных окнах поверх основного интерфейса
- ✅ **Визуальные индикаторы загрузки** — анимированный спиннер + значки загрузки на кнопках отчетов
- ✅ **Параллельная загрузка данных** — все 3 отчета загружаются одновременно при нажатии "ОБНОВИТЬ ДАННЫЕ"
- ✅ **Валидация данных** — проверка наличия данных перед открытием модалок (отображение "Данные не загружены")
- ✅ **Удобное закрытие модалок** — клик вне тела модального окна автоматически его закрывает

### Доступ:
Страница доступна по адресу `/fin-report` (требуется авторизация)

### Использование:

#### 🆕 Новый рабочий процесс (Modal Workflow):

**1. Кнопка "📊 ОБНОВИТЬ ДАННЫЕ"** (верхний правый угол, фиолетовая):
   - Нажатие запускает параллельную загрузку всех 3 отчетов с Wildberries API
   - Появляется анимированный спиннер с текстом "Загрузка данных..."
   - На каждой кнопке отчета отображается значок ⏳ с пульсацией
   - По мере загрузки каждого отчета его индикатор ⏳ исчезает
   - После загрузки всех отчетов блок загрузки автоматически скрывается

**2. Настройка параметров** (до или после загрузки):
   - Добавьте компании через кнопку "🏢 Управление компаниями" (каждая с отдельным API ключом)
   - Выберите компанию в селекторе или "🌐 Все активные компании"
   - Выберите период отчета (по умолчанию последние 30 дней)

**3. Просмотр отчетов** (после загрузки данных):
   - **Кнопка "📈 ФИН ОТЧЁТ"** — открывает модальное окно с полным финансовым отчетом (82 колонки)
   - **Кнопка "💰 ПРОДАЖИ"** — открывает модальное окно с отчетом по продажам (сгруппировано по артикулам)
   - **Кнопка "📦 ЗАКАЗЫ"** — открывает модальное окно с детализацией заказов
   - **Кнопка "📦 ЗАКАЗЫ"** — открывает модальное окно с детализацией заказов
   - **Кнопка "💰 СЕБЕСТОИМОСТЬ"** — управление себестоимостью товаров (см. ниже)
   - Если данные еще не загружены — в модалке отобразится сообщение "Данные не загружены"

### 🆕 Модуль себестоимости товаров

**Назначение:** Учет себестоимости товаров для точного расчета маржинальности.

**Функционал:**
- ✅ **Автоматическая загрузка товаров** — при открытии модалки подгружаются все товары из выбранного периода
- ✅ **Фотографии товаров** — миниатюры 50×50px загружаются автоматически из CDN WB
- ✅ **5 колонок таблицы**:
  - 📸 **Фото** — изображение товара с fallback на иконку 📦
  - **Бренд** — бренд товара из WB
  - **Артикул WB** — nmId товара
  - **Название товара** — редактируемое поле для пользовательского названия
  - **Себестоимость (₽)** — редактируемое поле для ввода себестоимости
- ✅ **Умная кнопка сохранения**:
  - По умолчанию неактивна (серая)
  - Активируется при вводе данных в любое поле
  - После успешного сохранения снова становится неактивной
- ✅ **Хранение в БД** — данные сохраняются в таблицу `product_costs` с привязкой к компании
- ✅ **Автозагрузка сохранённых данных** — при повторном открытии подгружаются ранее введённые значения

**Использование:**
1. Откройте модалку "💰 Себестоимость"
2. Таблица автоматически заполнится товарами из текущего периода
3. Введите пользовательские названия и себестоимость для нужных товаров
4. Нажмите "💾 Сохранить" (кнопка активируется автоматически)
5. Данные сохраняются в БД и будут доступны при следующем открытии

**4. Работа с модальными окнами**:
   - Модалка открывается поверх основного интерфейса
   - Таблица внутри модалки с sticky header (заголовок остается видимым при прокрутке)
   - Кнопка "✖" в правом верхнем углу для закрытия
   - Клик за пределами модального окна (на затемненном фоне) также закрывает модалку
   - Только одна модалка может быть открыта одновременно

**5. Индикаторы состояния**:
   - **Во время загрузки**: Блок "Загрузка данных..." + пульсирующие ⏳ на кнопках
   - **После загрузки**: Индикаторы исчезают, кнопки готовы к использованию
   - **Нет данных**: Сообщение "Данные не загружены" в модалке (серый цвет)

### Преимущества Modal Workflow:
- 🚀 **Быстрая параллельная загрузка** — все отчеты загружаются одновременно
- 👁️ **Четкая обратная связь** — визуальные индикаторы показывают прогресс загрузки
- 🎯 **Фокус на данных** — модалки концентрируют внимание на одном отчете
- ⚡ **Быстрое переключение** — просмотр разных отчетов без перезагрузки
- 💡 **Интуитивность** — понятно когда данные загружаются, когда готовы, когда отсутствуют

### Мультикомпания в отчетах:
- **Продажи**: При выборе "Все активные компании" данные агрегируются со всех компаний
  - Добавляется колонка "Компания" (первая колонка)
  - Сортировка по колонкам: клик на заголовок (↕ индикатор)
  - По умолчанию отсортировано по компании (алфавитно)
- **Финансовый отчёт**: При выборе "Все активные компании" появляются табы
  - Каждая компания в отдельном табе
  - Клик по табу переключает данные
  - Активный таб подсвечен (белый фон, фиолетовый текст, нижняя граница)
